Field Type Required Description
name string Yes Full display name
email string Yes Unique email address
password string Yes Min 8 characters
role string No admin | analyst

📊
Financial Records
/api/records
POST /records/create-record Add a new income or expense record ▼
Required Role Admin
Request Body
JSON
{
  "amount": 1500,
  "recordType": "income", // or "expenditure"
  "category": "Salary",
  "date": "2026-04-05",
  "description": "Monthly paycheck"
}
Field Reference
Schema
recordType: "income" | "expenditure"
amount:     number  // positive value
category:   string  // e.g. Salary, Travel
date:       string  // YYYY-MM-DD
description:string  // optional notes
GET /records/get-all-records Fetch records with filtering & sorting ▼
Query Parameters
Parameter Type Required Description
type string No income | expenditure
category string No Filter by category, e.g. Travel
startDate string No Format: YYYY-MM-DD
endDate string No Format: YYYY-MM-DD
sortBy string No Field to sort by, e.g. amount
order string No asc | desc — Default: desc
PATCH /records/update-record/:id Partially update an existing record ▼
Required Role Admin
Request Body
JSON
// Any subset of record fields
{
  "amount": 2000,
  "category": "Freelance"
}
Path Parameters
Params
:id  // MongoDB ObjectId of the record
     // e.g. 67a1b2c3d4e5f6789abc

User Management
/api/users · Admin only
PATCH /users/:id/toggle-status Toggle active / inactive status ▼
Required Role Admin
Success Response
200 OK
{
  "success": true,
  "user": {
    "_id": "...",
    "status": "inactive"
  },
  "message": "User status changed to inactive"
}
PATCH /users/:id Update user profile or role ▼
Required Role Admin
Request Body
JSON
{
  "role": "analyst",
  "name": "Updated Name"
}
Available Roles
Schema
"admin"    // Full CRUD access
"analyst"  // Read + dashboard access

Dashboard Analytics
/api/dashboard
GET /dashboard/summary Financial totals and net balance ▼
Required Role Admin Analyst
Success Response
200 OK
{

  "summary": {
    "totalIncome": 5000,

Required Role Admin Analyst
Success Response
200 OK
{

⚠️
Error Responses
Standardized error format
Standard Error Envelope
{
  "success": false,
  "message": "Detailed error message here"
}
HTTP Status Codes
400
Bad Request

Missing required fields or validation failure in the request body.

401
Unauthorized

Missing or invalid Bearer token in Authorization header.

403
Forbidden

Authenticated user lacks the required role (RBAC enforcement).

404
Not Found

Resource not found at the specified ID or endpoint path.

500
Internal Server Error

Unexpected failure on the server side. Check server logs.
